# Source Code Directory (`src/`)

このディレクトリには、Synaptic Routing Architecture (SRA) のモデル定義、実験スクリプト、および各種ユーティリティが含まれています。

## ファイル構成

### 1. モデル定義 (Model Architectures)

* **`sra_reference.py`**
  * **役割**: オリジナルのSRA（リファレンスモデル）と、比較検証用のベースラインモデル（Transformer, MLP）が定義されているファイル。
  * **用途**: アルゴリズムの純粋な数学的挙動を理解したり、新しいアイデアをテストしたりするための「分かりやすさ重視」の実装です。
* **`sra_gpu_models.py`**
  * **役割**: GPU (MPS/CUDA) での実行効率を極限まで高めた最適化済みSRAモデルが定義されているファイル。
  * **用途**: `BatchedSRAModel`（並列ゴリ押し版）、`MoESRAModel`（Attentionを共有した真のスパース最適化版）、`SeqSRAModel`（系列単位のルーティング版）が含まれています。大規模な学習や高速化が必要な場合はこちらのモデルを使用します。

### 2. 学習・実験スクリプト (Training & Experiments)

* **`train_moe_sra.py`**
  * **役割**: 最適化アプローチとして公式採用された **`MoESRAModel` だけを単独で学習** するための独立したスクリプト。
  * **用途**: SRAの最速・最高精度のモデルをシンプルに学習させたい場合や、MoEアーキテクチャの学習フローをコードで追いかけたい場合に使用します。
* **`sra_experiment.py`**
  * **役割**: リファレンスモデルを含むあらゆるモデル（SRA, Transformer, MLPなど）を検証するための汎用的な実験スクリプト。
  * **用途**: `train_moe_sra.py` から呼び出されるバッチ生成やロス計算などの共通ユーティリティ関数もここに定義されています。
* **`compare_architectures.py`**
  * **役割**: 複数のモデルアーキテクチャ間でベンチマーク（速度、VRAM消費、精度など）を自動計測して比較するスクリプト。
  * **用途**: 新しいモデルを実装した際に、ベースラインと比べてどれだけ効率的かを定量評価するために使用します。

### 3. 共通・ユーティリティ (Utilities)

* **`constants.py`**
  * **役割**: 語彙サイズ（`VOCAB_SIZE`）、特殊トークン（`PAD`, `BOS`, `EOS` など）、タスクの設定などの「定数群」を一元管理しているファイル。
  * **用途**: すべてのモデルやスクリプトから共通でインポートして使用されます。

## 開発フローのおすすめ

1. **アーキテクチャの理解**: まず `sra_reference.py` の `SRAModel` と `TinySynapse` の構造を読んでください。
2. **モデルの最適化**: 構造を理解したら `sra_gpu_models.py` の `MoESRAModel` を見て、どのように高速化されているかを比較してください。
3. **学習の実行**: 実際の学習は `python src/train_moe_sra.py --task reverse` などを実行して挙動を確認してください。
