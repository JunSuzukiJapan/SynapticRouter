# 알고리즘 추론에서의 다중 작업 학습 및 라우팅 분석

## 개요
Synaptic Routing Architecture (SRA)가 작업 특성에 따라 자율적으로 전문가(시냅스)를 할당하고 특성화(모듈화)를 달성할 수 있는지 검증했습니다.
서로 다른 특성을 가진 4개의 알고리즘 추론 작업(`copy`, `reverse`, `paren`, `addmod`)에 대해 단일 SRA 모델을 동시에 학습시키고 그 추론 결과와 라우팅 분포를 분석했습니다.

## 작업 세부 정보
시퀀스 시작 부분에 작업 지정을 위한 특수 토큰을 추가하여 모델에 해결해야 할 작업을 지시합니다.
- `COPY`: 입력 시퀀스를 그대로 출력합니다.
- `REVERSE`: 입력 시퀀스를 역순으로 출력합니다.
- `PAREN`: 괄호가 일치하는지 확인합니다 (`Y` 또는 `N`).
- `ADDMOD`: 두 숫자를 더한 값의 일의 자리(나머지)를 계산합니다.

## 학습 결과
10,000단계의 동시 학습 후 모든 작업에서 완벽하게 정확한 추론 결과(약 100% 정확도)를 얻었습니다. 이는 SRA가 작업 간의 간섭(치명적 망각)을 방지하면서 여러 작업을 단일 모델에 통합할 수 있음을 보여줍니다.

## 라우팅 분석 (코사인 유사도)
학습된 모델에 각 작업에 대한 데이터를 입력할 때 각 레이어에서 사용된 시냅스(전문가) 분포를 추출하고 작업 간의 코사인 유사도를 계산했습니다.

### Layer 0 (얕은 레이어)
```text
                copy   reverse     paren    addmod
      copy     1.000     0.708     0.788     0.867
   reverse     0.708     1.000     0.836     0.299
     paren     0.788     0.836     1.000     0.609
    addmod     0.867     0.299     0.609     1.000
```
얕은 레이어에서는 일반적인 시냅스가 모든 작업에서 어느 정도 공유됩니다. 이는 토큰의 기본 특징 추출이 교차 작업으로 수행됨을 시사합니다.

### Layer 1 (깊은 레이어)
```text
                copy   reverse     paren    addmod
      copy     1.000     0.969     0.213     0.336
   reverse     0.969     1.000     0.029     0.134
     paren     0.213     0.029     1.000     0.858
    addmod     0.336     0.134     0.858     1.000
```
깊은 레이어에서는 작업별 라우팅 분포가 명확하게 그룹화되었습니다.

1. **시퀀스 작업 그룹**:
   `COPY` 및 `REVERSE`는 **0.969**의 매우 높은 유사도를 나타냈습니다. 둘 다 "입력 시퀀스의 순서를 유지하거나 역전시키면서 재구성하는" 작업이며 SRA는 이를 유사한 속성을 가진 작업으로서 동일한 시냅스 그룹으로 라우팅합니다.

2. **계산/논리 그룹**:
   `PAREN` 및 `ADDMOD`는 언뜻 보기에 관련이 없어 보이지만 유사도가 **0.858**로 높았습니다. 둘 다 "상태 관리(괄호 깊이)" 또는 "산술 연산(덧셈)"과 같이 단순한 시퀀스 재배열 이상의 논리적 계산을 요구하므로 동일한 시냅스 그룹으로 라우팅됩니다.

3. **그룹 간의 명확한 분리**:
   위의 두 그룹 간의 유사도는 **0.029 ~ 0.336**으로 극히 낮습니다.

## 결론
SRA 라우터가 **인간의 명시적인 지시 없이 작업의 특성을 자율적으로 파악하고 유사한 작업에 대해 시냅스를 공유하며 서로 다른 작업에 대해 명확하게 다른 시냅스를 사용하도록 모듈화**한다는 것이 정량적으로 입증되었습니다.
