# SRA階層型ルーターにおけるHard Routing手法の比較検証

## 1. 実験の目的
SRA（Synaptic Routing Architecture）をシナプス単位ではなく、モデル単位に拡張した「上位ルーター（Super Router / Hierarchical Routing）」において、複数モデルを統合する際の最適経路選択（Routing）手法を検証する。
特に、不要なモデルの計算を完全に省略（計算量削減）するための **Hard Routing（Top-1 Gating）** において、どの数理的手法が最も効果的に「自信を持った確率分布（1.0 vs 0.0）」を獲得できるかを調査した。

## 2. 比較手法
以下の3つのルーティング手法を実装し、フランス語・ドイツ語・スペイン語の翻訳タスク（Model 1 = 仏・独, Model 2 = 西）で比較した。

1. **SOFT (通常のSoftmax)**: 確率的な重み付けを行い、すべてのモデルの出力をアンサンブルする。
2. **STE (Straight-Through Estimator)**: 順伝播ではTop-1を強制選択し、逆伝播ではSoftmaxの勾配をそのまま流すハック的手法。
3. **GUMBEL (Gumbel-Softmax)**: Softmaxにガンベルノイズを加え、微分可能な疑似ハードルーティングを行う手法。

## 4. 考察

* **SOFTの限界 (Lazy Routing)**: 
  文脈を理解して正しいモデルに高い確率を割り当てることには成功しているが、依然として5〜9%の確率が不要なモデルに割り当てられている。推論時に不要なモデルの計算を完全にカットすることができない。
* **STEの限界 (確率分布の濁り)**: 
  Hard RoutingとしてTop-1を選択することには成功している。しかし、正解のモデルを選べるようになった時点で誤差が減少し、それ以上確率を尖らせる（1.0に近づける）ための強い勾配が発生しなくなる。結果として `0.6 vs 0.4` という自信のない濁った確率分布のまま学習が収束してしまった。
* **Gumbel-Softmaxの優位性**: 
  学習時にガンベルノイズが加わることで、ルーターは「ノイズが乗っても確実に正しいモデルを選べるように」確率分布を極端に尖らせるよう強制される。その結果、完璧な `1.0 vs 0.0` のスパースルーティングを獲得できた。

## 5. 結論
独立した複数モデルを束ねる上位ルーター（モデルレベルのMoE）を実装し、計算リソースを動的かつ完全に切り替えるためには、**Gumbel-Softmaxを用いたTop-1 Gating** が最も安定的かつ理想的な手法であることが証明された。
