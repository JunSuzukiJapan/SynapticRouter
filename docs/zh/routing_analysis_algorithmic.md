# 算法推理中的多任务学习和路由分析

## 概述
我们验证了突触路由架构（Synaptic Routing Architecture, SRA）是否能够根据任务性质自主分配专家（突触）并实现专业化（模块化）。
我们同时在一个单一的 SRA 模型上训练了四个具有不同特征的算法推理任务（`copy`, `reverse`, `paren`, `addmod`），并分析了其推理结果和路由分布。

## 任务详情
通过在序列开头添加一个用于任务规范的特殊 token，我们指示模型要解决哪个任务。
- `COPY`: 原样输出输入序列。
- `REVERSE`: 反向输出输入序列。
- `PAREN`: 判断括号是否匹配（`Y` 或 `N`）。
- `ADDMOD`: 计算两个数字相加的个位数（取模）。

## 训练结果
在进行了 10,000 步的同时训练后，我们在所有任务中都获得了完全准确的推理结果（接近 100% 的准确率）。这表明 SRA 能够将多个任务集成到单个模型中，同时防止任务之间的干扰（灾难性遗忘）。

## 路由分析 (余弦相似度)
当将每个任务的数据输入到经过训练的模型中时，我们提取了每层中使用了哪些突触（专家）的分布，并计算了任务之间的余弦相似度。

### Layer 0 (浅层)
```text
                copy   reverse     paren    addmod
      copy     1.000     0.708     0.788     0.867
   reverse     0.708     1.000     0.836     0.299
     paren     0.788     0.836     1.000     0.609
    addmod     0.867     0.299     0.609     1.000
```
在浅层中，所有任务在一定程度上共享共同的突触。这表明 token 的基本特征提取是跨任务执行的。

### Layer 1 (深层)
```text
                copy   reverse     paren    addmod
      copy     1.000     0.969     0.213     0.336
   reverse     0.969     1.000     0.029     0.134
     paren     0.213     0.029     1.000     0.858
    addmod     0.336     0.134     0.858     1.000
```
在深层中，每个任务的路由分布被明确地分组。

1. **序列操作组**:
   `COPY` 和 `REVERSE` 表现出非常高的相似度，达到 **0.969**。这两个都是“在保持或反转输入序列顺序的同时重构输入序列”的任务，SRA 将它们路由到相同的一组突触，作为具有相似属性的任务。

2. **计算/逻辑组**:
   `PAREN` 和 `ADDMOD` 乍一看不相关，但它们的相似度很高，达到 **0.858**。这两个任务都需要超越简单序列重新排列的逻辑计算，例如“状态管理（括号深度）”或“算术运算（加法）”，因此它们被路由到相同的一组突触。

3. **组间明确的分离**:
   上述两组之间的相似度极低，为 **0.029 到 0.336**。

## 结论
定量证明了 SRA 路由器 **无需人类明确指示即可自主辨别任务的性质，为相似任务共享突触，并模块化以明确地为不同任务使用不同的突触**。
