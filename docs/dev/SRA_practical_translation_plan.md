# SRA 多言語機械翻訳 実務レベル検証・実験計画

## 背景と目的
現在の小規模な翻訳実験（Toyデータセット）を**「実務レベル・実用レベルの翻訳」にスケールアップ**します。
その過程で、本アーキテクチャの核心である**「SRAのシナプスが言語ごとに効率的に割り当てられるか（ルーティングの分離と共有）」**を定量的・視覚的に検証することを目的とします。

## ターゲット環境（重要）
- **実行環境**: 本格的なGPU環境（別のMac、あるいは大容量VRAMを搭載したマシン）での実行を想定。
- 長時間（数時間〜数日）の学習プロセスに耐えうるように、チェックポイント機能や最適化手法をスクリプトに組み込みます。

## 実装計画

### 1. データロード・パイプラインの本格化
これまではスクリプト内に直書きされたToyデータを使用していましたが、実務レベルのパラレルコーパスを動的にロードしてバッチ処理する仕組みを構築します。

* **新規ファイル**: `src/data_loader_translation.py`
  * Hugging Face の `datasets` ライブラリを使用し、`iwslt2017`（または `opus100`）といった中〜大規模の多言語対訳データセットをロードします。
  * メモリに載りきらない大規模データをストリーミング（イテラブル）で読み込む機能を実装します。
  * SentencePiece / BPE ベースの適切な多言語トークナイザ（例：Hugging Face 上の Llama または mBART トークナイザ）を用いた前処理を行います。
  * **対象言語**: 英語(English)、日本語(Japanese)、フランス語(French) の3言語をベースとします。

### 2. 学習スクリプトのスケーリングと最適化
実用レベルの大規模学習に耐えるよう、AMP（自動混合精度）などを導入した本格的なトレーニングスクリプトを作成します。

* **新規ファイル**: `src/train_translation_large.py`
  * `torch.cuda.amp.autocast` / `torch.autocast` を用いた高速化とVRAM節約。
  * `torch.compile` によるモデルコンパイル対応（PyTorch 2.x向け）。
  * チェックポイント（モデルの重みとオプティマイザのステート）の定期保存と再開機能。
  * 定期的なバリデーション（Validation）による過学習の監視とロギング（標準出力ベース）。
* **修正対象**: `src/sra_language_models.py`
  * 大規模学習時に安定するよう、必要に応じて初期化やDropoutなどの微調整を行います（基本のルーティングロジックは維持）。

### 3. ルーティング・シナプス効率の分析
「SRAのシナプスが言語や構文で効率的に切り替わるか」を定量評価します。

* **新規ファイル**: `src/analyze_translation_routing.py`
  * 学習済みモデルを読み込み、特定のテスト文（「英語の主語」「日本語の動詞」など）を入力した際に、どのシナプス（エキスパート）が強く発火しているかを可視化します。
  * 言語間の「シナプス利用の重複度」をコサイン類似度などで計算し、「英語とフランス語（SVO）は共通のシナプスを使い、日本語（SOV）は別のシナプスを使っているか」といった仮説を検証します。

## 検証プラン
1. **Automated Tests / Validation**: 複数の言語ペアに対するバリデーションロスおよび簡易的な翻訳精度を計算し、学習の進捗をトラッキングします。
2. **Manual Verification**: 
   * `analyze_translation_routing.py` を実行し、生成されたヒートマップ等から論理的にシナプスが使い分けられているかを確認します。
   * 推論モードを用いて手動でテスト文を入力し、実務レベルに近づいていることを目視で確認します。

## 実行ガイド（別のMac / GPU環境用）

本計画に基づいて実装されたスクリプトを別の環境で実行する手順です。

### 1. 環境セットアップ
新しい環境でプロジェクトをクローン後、必要なライブラリをインストールします。
データローダーとロギング、可視化のために以下のパッケージが必要です。

```bash
pip install datasets tiktoken wandb matplotlib seaborn
```

### 2. 大規模学習の実行
AMP（自動混合精度）やコンパイル（PyTorch 2.x以降）を有効にして、学習をスタートします。
`train_translation_large.py` は自動的にデータをダウンロード（ストリーミング）し、チェックポイントを保存しながら学習を進めます。

```bash
# AMPとモデルコンパイルを有効にし、バッチサイズ64で20,000ステップ学習する例
python src/train_translation_large.py \
    --amp \
    --compile \
    --batch-size 64 \
    --steps 20000 \
    --save sra_translation_large.pt
```
※ `--wandb` フラグをつけることで、Weights & Biases にログを送信できます。
※ 中断した場合は `--resume` フラグをつけることでチェックポイントから再開可能です。

### 3. ルーティングの分析・可視化
学習が完了したら（あるいはチェックポイントができたら）、モデルを読み込んで「言語ごとにどのシナプスが発火しているか」を分析します。

```bash
python src/analyze_translation_routing.py --model-path sra_translation_large.pt
```

実行後、`translation_routing_analysis.png` という画像ファイルが生成され、各レイヤーにおける英・日・仏のシナプス利用確率がヒートマップ（棒グラフ）として可視化されます。
