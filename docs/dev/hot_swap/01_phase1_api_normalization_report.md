# フェーズ1: シナプスAPI規格化（正規化）の検証レポート

## 1. 背景と目的
SRA（Synaptic Routing Architecture）において、将来的に未知のシナプスを「安全にHot-Swap（動的入れ替え）」するためには、シナプスが出力するベクトルのスケール（長さ・分散）が異常値になり、ルーターや後続ネットワークを崩壊させる事故を防ぐ必要があります。
このため、「どのシナプスを通っても出力ベクトルのスケールが一定になる」という**安全なAPI規格**を設けるべく、シナプス出力への強制的な正規化（Normalization）が学習およびルーティング（専門化）に与える影響を検証しました。

## 2. 実験設定
ベースアーキテクチャ: `CausalMoESRABlock`
タスク: マルチドメイン学習（CODE, MATH, TEXT） / 400 Steps
比較対象:
1. **ベースライン (正規化なし)**: 通常のMLP出力。
2. **ピュアLayerNorm**: 学習可能なアフィンパラメータを持たない純粋なLayerNorm（`F.layer_norm`）。出力の平均を0、分散を1に強制。
3. **L2正規化**: ベクトルの長さを強制的に1に揃え、モデル次元数の平方根（`math.sqrt(dim)`）を掛けることで適切な分散にスケーリング（`F.normalize(p=2)`）。

## 3. 実験結果

| 手法 | 最終Loss | CODEの主要シナプス | MATHの主要シナプス | TEXTの主要シナプス | 評価 |
| :--- | :--- | :--- | :--- | :--- | :--- |
| **ベースライン** | 0.0867 | [11, 13, 12] | [ 1,  9, 11] | [ 0, 11,  3] | 基準。明確な使い分けが確認できる。 |
| **LayerNorm** | 0.0815 | [ 0, 14, 15] | [ 0, 15,  8] | [ 8,  0, 15] | 学習は成功するが、ドメイン間でのシナプスの重複（相乗り）が増加し、専門化がやや曖昧に。 |
| **L2正規化** | **0.0667** | **[10,  7,  0]** | **[14, 11,  3]** | **[11, 13, 12]** | **Lossが最も低く収束**。かつドメインごとのシナプスの**専門化が極めて鮮明**に現れた。 |

## 4. 結論と今後の指針
**「シナプス出力へのL2正規化」は、Hot-Swapの安全性を担保する最強のAPIプロトコルである。**
出力の長さを固定するルールを課すことは、表現力を奪って学習を阻害するどころか、ルーターの予測を安定させ、**モデル全体の学習効率（Lossの低下）を向上させる効果**があることが判明しました。

これにより、未知のプラグイン（外部で学習されたシナプス）をガチャンと組み込んでも、出力がL2正規化される限り、ルーターが予測不可能な巨大ベクトルによってパニックを起こす物理的なリスクが排除されます。
SRAのシナプス規格として、この **「L2正規化 + sqrt(dim) スケーリング」を標準プロトコルとして採用** します。
