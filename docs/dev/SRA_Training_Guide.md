# SRA (Synaptic Routing Architecture) 学習ガイド

SRAは、動的にサブネットワーク（シナプス）を選択して計算を行うスパースなアーキテクチャです。そのため、通常のTransformerやMLPとは異なり、**ルーティングの学習とシナプスの専門化**をうまく両立させるための特殊な学習フェーズとハイパーパラメータの設定が必要になります。

このドキュメントでは、SRAの学習を成功させるための重要なパラメータと、学習フェーズの役割について解説します。

---

## 1. 4つの学習フェーズ（カリキュラム学習）

SRAでは、シナプスが適切に専門化（役割分担）し、一部のシナプスだけが過剰に使われる「モード崩壊（Mode Collapse）」を防ぐために、4つのフェーズに分けて学習を行います。

| フェーズ | 期間の目安 | 概要と目的 |
|---|---|---|
| **1. Warmup** | 全体の10% | ルーターを無視して**全シナプス（Dense）で計算**します。初期状態で一部のシナプスが全く選ばれず「死んだシナプス（Dead Experts）」になるのを防ぎ、すべての重みに初期の勾配を与えます。 |
| **2. Joint** | 全体の70% | ルーターとシナプスを同時に学習します。ルーターは担当するシナプスを選び始め、シナプスはその予測精度を上げようとします。この時、**ロードバランス損失（Load Balance Loss）**をかけて使用頻度を均等化します。 |
| **3. Stabilize** | 全体の15% | **ルーターの重みを凍結（Freeze）**し、シナプスのみを学習させます。ルーターの選択が固定されることで、「動く的（Moving Target）」問題が解消され、シナプスが自分が担当するトークンに特化（Specialization）しやすくなります。 |
| **4. Specialize** | 全体の5% | さらにシナプスの専門化を促すため、エントロピーを最大化するような損失（Specialization Loss）を追加し、よりシャープなルーティングを定着させます。 |

---

## 2. 重要なハイパーパラメータと損失関数

学習を成功させるためには、以下の損失関数の重み付けとパラメータ設定が重要です。

### `load_balance` (推奨: 0.1 〜 0.5)
- **役割**: 一部の優秀なシナプスばかりが選ばれるのを防ぐためのペナルティです。ルーターの出力確率が均等（Uniform）になるように働きます。
- **調整のコツ**: 値が大きすぎるとルーティングがランダムに近づき精度が落ちます。小さすぎると1〜2個のシナプスしか使われなくなります。タスクの複雑さに応じて調整します。

### `k` (Top-kの選択数) (推奨: 2)
- **役割**: 1回の計算でいくつのシナプスを同時に動かすか（合成するか）を決定します。
- **調整のコツ**: `k=1` は最も処理が速くスパースですが、学習が不安定になりやすいです。`k=2` にすることで、シナプス間の知識の補完が行われ、学習の安定性と精度が劇的に向上します（MoEでも標準的な設定です）。

### `warmup_steps` (推奨: 総ステップの5%〜10%)
- **役割**: 初期化時にすべてのシナプスを満遍なく育てる期間です。
- **調整のコツ**: これが短すぎると「使われないシナプス」が発生しやすくなります。長すぎると、計算コスト（Dense計算）が無駄にかかります。

### `self_gen_weight` (自己生成ロス) (推奨: 0.5)
- **役割**: 自己回帰的な予測能力を高めるための正則化として機能します。Teacher Forcingだけでなく、モデル自身が生成した系列（Prefix）に対するLossも計算に加えます。
- **調整のコツ**: 特に系列タスクにおいて、汎化性能を高める効果があります。

---

## 3. 初期化の工夫 (Orthogonal Initialization)

ルーターの初期化には、ランダムなガウス分布ではなく **直交初期化（Orthogonal Initialization）** を用いています。
さらに、埋め込み次元（`dim`）の平方根でスケーリングすることで、学習の初期段階で特定のシナプスのロジットが極端に大きくならず、すべてのシナプスが選ばれるチャンスを均等に与える工夫をしています。

---

## 4. トラブルシューティング

- **Q. 全然Lossが下がらない**
  - `warmup_steps` を少し長めに取り、初期に全シナプスが適切に学習できているか確認してください。
  - ルーターのスケール（温度パラメータ）や学習率（`lr`）を見直してください。

- **Q. 特定のシナプスしか使われていない (Usage Entropy が 0 に近い)**
  - `load_balance` の重みを上げてください。
  - `k=1` で学習している場合は、`k=2` に上げてみてください。

- **Q. 学習終盤で精度が急に落ちる**
  - ルーターの凍結（Stabilizationフェーズ）がうまく機能していない可能性があります。ルーターの重みが確実に Freeze されているか確認してください。
