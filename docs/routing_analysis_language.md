# マルチタスク言語モデリングにおけるルーティング分析 (Code / Math / Text)

## 概要
Mixtral等で採用されているSparse MoEアプローチを参考にし、SRA（Synaptic Routing Architecture）を用いて語彙・文法の異なる3つのドメイン（Code, Math, Text）を同時に学習させる「異種ドメイン言語モデリング」実験を実施した。本ドキュメントは、その学習結果、推論例、およびシナプスのルーティング（専門化）に関する予備的な分析結果をまとめたものである。

## 実験設定
- **タスク構成**: 
  - `Code`: Pythonコードスニペット
  - `Math`: LaTeX形式の数式
  - `Text`: 日本語・英語の混ざった自然言語
- **トークン化**: 文字レベル（Char-level Tokenization、語彙サイズ=178）
- **モデル**: `MoESRALanguageModel` (Decoder-only 自己回帰言語モデル)
- **ハイパーパラメータ**: 
  - Dimensions: 64
  - Layers: 2
  - Number of Synapses: 16
  - k (Routing Top-K): 2
  - Sequence Length: 32
  - Steps: 1000 steps

## 1. 推論品質の評価
1000ステップの学習後、各ドメインの一部をプロンプトとして与え、Temperature Sampling (T=0.7) を用いて推論を行った結果、各ドメインの文字列パターンを一部再現する例が確認された。

### [Code] タスク
**プロンプト**:
```python
is {i}")

def ad
```
**生成結果**:
```python
is {i}")

def add(a):
    is lefthvead aleanga
```
**分析**: `def ad` から `def add(a):` に近い関数定義の形を生成し、Pythonコードらしいインデントも一部維持している。ただし後続の文字列には崩れがあり、コード生成能力としては限定的な予備結果である。

### [Math] タスク
**プロンプト**:
```latex
frac{P(B|A)P(A)}
```
**生成結果**:
```latex
frac{P(B|A)P(A)}{P(B)}

\frac{\pac{\iacosen)
```
**分析**: ベイズの定理の分子 `frac{P(B|A)P(A)}` を受けて、分母 `{P(B)}` を補完する例が見られた。また、改行後に `\frac{` というLaTeX特有のコマンド形式を継続しようとする振る舞いも見られた。

### [Text] タスク
**プロンプト**:
```text
ght-iron lattice
```
**生成結果**:
```text
ght-iron lattice tower on the Chasngelampff la
```
**分析**: "wrought-iron lattice" に続くエッフェル塔説明文に近い断片を再生成しており、学習データ中の局所的な文字列パターンを保持している可能性がある。

## 2. ルーティングとシナプス分化の分析
学習初期（Early）と学習終盤（Final）における、全16個のシナプスの使用頻度（Usage）を比較し、ドメイン間での棲み分け状況を検証した。

### シナプス使用頻度の推移
| Task | 初期 (Early Usage) | 最終 (Final Usage) | 特化しているシナプス |
| :--- | :--- | :--- | :--- |
| **Code** | 広く分散 (Max 18%) | **[8]: 30%**, **[15]: 16%**, **[11]: 13%** | シナプス 8 |
| **Math** | 広く分散 (Max 19%) | **[10]: 33%**, **[13]: 24%**, **[11]: 12%** | シナプス 10, 13 |
| **Text** | 広く分散 (Max 24%) | **[15]: 20%**, **[11]: 18%**, **[0]: 16%** | シナプス 0, 15 |

### 分析考察
1. **初期状態（Warmup時）の均一化**
   学習初期はロードバランシング損失により、各タスクが16個のシナプスを比較的均等（各0.05～0.15程度）に使用していた。
2. **ルーティングの凍結と分化**
   特定のステップでルーターをフリーズし（Phase transition）、各シナプスへの専門化を促した結果、ドメインごとに使用される主要なシナプスが明確に分離した。
   - `Code` の処理には **シナプス8** が支配的に使われるようになった。
   - `Math` の処理は **シナプス10と13** が主に担当するようになった。
   - `Text` の処理は **シナプス0と15** に依存するようになった。
3. **干渉の回避（Catastrophic Forgettingの防止）**
   異なる文法規則（Pythonのインデント、LaTeXのバックスラッシュ記法、自然言語の文脈）を持つ3つのドメインを同一ネットワークで学習できた背景として、**MoEによるシナプスの機能分化**が寄与している可能性がある。ただし、この解釈を強く主張するには、複数seed、密なベースライン、ルーター固定/ランダムルーターとの比較が必要である。

## 結論
SRAを用いた多ドメイン自己回帰言語モデリングにおいて、少数のパラメータとステップ数でも、ドメインごとに異なるシナプス使用分布が形成される傾向が観測された。今後は、生成品質の定量評価、ベースライン比較、seed違いでの再現性確認を行い、SRA/MoEアーキテクチャが異種データ混合学習にどの程度有効かをより厳密に評価する。
