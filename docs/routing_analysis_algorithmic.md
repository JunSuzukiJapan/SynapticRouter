# アルゴリズム的推論におけるマルチタスク学習とルーティング分析

## 概要
Synaptic Routing Architecture (SRA) がタスクの性質に応じて自律的にエキスパート（シナプス）を割り当て、専門化（モジュール化）を実現できるかを検証しました。
単一の SRA モデルに対して、性質の異なる4つのアルゴリズム的推論タスク（`copy`, `reverse`, `paren`, `addmod`）を同時に学習させ、その推論結果とルーティングの分布を分析しました。

## タスク詳細
系列の先頭にタスク指定用の特殊トークンを付与することで、モデルに解くべきタスクを指示します。
- `COPY`: 入力系列をそのまま出力する。
- `REVERSE`: 入力系列を反転して出力する。
- `PAREN`: 括弧の対応が取れているかを判定する（`Y` or `N`）。
- `ADDMOD`: 2つの数字の足し算の1の位（剰余）を計算する。

## 学習結果
10,000 ステップの同時学習を行った結果、この実験設定ではすべてのタスクにおいて Accuracy 100% 付近の推論結果が得られました。これは、小規模なアルゴリズムタスクにおいて、SRA がタスク間の干渉を抑えながら複数タスクを単一モデルに統合できる可能性を示す予備結果です。

## ルーティングの分析 (Cosine Similarity)
学習済みモデルに対し、各タスクのデータを入力した際に、各レイヤーでどのシナプス（エキスパート）が使われたかの分布を抽出し、タスク間のコサイン類似度を計算しました。

### Layer 0 (浅いレイヤー)
```text
                copy   reverse     paren    addmod
      copy     1.000     0.708     0.788     0.867
   reverse     0.708     1.000     0.836     0.299
     paren     0.788     0.836     1.000     0.609
    addmod     0.867     0.299     0.609     1.000
```
浅いレイヤーでは、全タスク間で共通のシナプスがある程度共有されています。これは、トークンの基本的な特徴抽出がタスク横断的に行われていることを示唆しています。

### Layer 1 (深いレイヤー)
```text
                copy   reverse     paren    addmod
      copy     1.000     0.969     0.213     0.336
   reverse     0.969     1.000     0.029     0.134
     paren     0.213     0.029     1.000     0.858
    addmod     0.336     0.134     0.858     1.000
```
深いレイヤーでは、タスクごとのルーティング分布が明確にグループ化されました。

1. **系列操作グループ**: 
   `COPY` と `REVERSE` は、類似度が **0.969** と非常に高くなりました。どちらも「入力系列の順番を保持または反転して再構築する」というタスクであり、SRA はこれらを似た性質のタスクとして同一のシナプス群にルーティングしています。
   
2. **計算/論理グループ**: 
   `PAREN` と `ADDMOD` は、一見無関係に見えますが、類似度は **0.858** と高くなりました。これらはどちらも「状態の管理（括弧の深さ）」や「算術演算（足し算）」といった、系列の単純な並び替え以上の論理的な計算を要するため、同じシナプス群にルーティングされています。

3. **グループ間の明確な分離**:
   上記の2つのグループ間の類似度は **0.029 〜 0.336** と極めて低くなっています。

## 結論
この小規模実験では、SRA のルーターがタスク種別に応じて異なるシナプス使用分布を形成し、類似したタスクではシナプスを共有し、異なるタスクでは使用分布を分ける傾向が観測されました。今後は複数seed、ベースライン比較、タスク数や系列長を変えたアブレーションで、この傾向の再現性を確認する必要があります。
