# SRA Project: Next Steps (次回やることリスト)

本ドキュメントは、これまでの実験と検証を踏まえ、SRA（Synaptic Routing Architecture）開発の次のステップを整理したものです。

## 1. 優先課題：高度なルーティングとアンラーニングの追求

### 🚀 1.1. Routing Fallback (再割り当て) の実装と検証
- **背景**: `16_lazy_routing_prevention_experiment` の結果、単なる Capacity Limit（溢れたトークンの切り捨て）では Dead Synapse の蘇生に繋がらないことが判明しました。また、SRAのデフォルトの Load Balance Loss はすでに優秀（Dead 11%）であることも確認されました。
- **タスク**: 
  - ルーターの容量上限を超えたトークンを、確率が2番目に高い空きシナプスへ強制的に転送（Fallback）するロジックを実装する。
  - これにより、Dead Synapse Ratio を「完全な 0%」にできるか、またモデル全体の精度が向上するかを検証するノートブックを作成する。

### 🧠 1.2. 「挿入と削除」のモジュール化による完全な忘却（Unlearning）の確立
- **背景**: 混ざり合ったモデルから特定の知識を後から「完全（0%）に消去」するアルゴリズムを追求するよりも、ノートブック6や10などで実証した「Hot-Swapの挿入側（独立したプラグインとしての学習と結合）」のパラダイムを突き詰める方がSRAの思想に合致しているという結論に至りました。
- **タスク**:
  - 最初からベースモデルを凍結し、追加のドメインを完全に独立した「追加シナプス（プラグイン）」として学習・結合するワークフローをSRAの標準とする。
  - これにより、不要になったドメインは「追加したシナプスごと物理的に抜き取る（削除する）」だけで、ベースモデルに一切のダメージを与えずに**必然的に100%完全な忘却**が達成できることを公式な仕様・ドキュメントとして明文化する。
