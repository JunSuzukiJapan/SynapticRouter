# SRA Project: Next Steps (次回やることリスト)

本ドキュメントは、これまでの実験と検証を踏まえ、SRA（Synaptic Routing Architecture）開発の次のステップを整理したものです。

## 1. 現在のステータス（デュアルトラック戦略への移行）
SRAプロジェクトは現在、以下の2つのアプローチ（デュアルトラック戦略）で並行して進行しています。
1. **ネイティブSRAアプローチ**: 独自の言語モデル内部に動的ルーティングをゼロから構築する基礎研究。（Lazy Routing問題の克服など）
2. **OSS LLM統合（マザーボード）アプローチ**: Llama 3などの既存LLMの規格に合わせ、再学習なしで異種モジュールを接続する実用化研究。（Notebook 23でPoC成功）

## 2. 優先課題：最適なルーターアーキテクチャの検証
Notebook 23のPoCにより、OSS LLM（Frozen）と超軽量ルーターを組み合わせた「ネイティブ統合学習」が成功しました。しかし、現在の「単層Linear ＋ 全トークン平均（Mean Pooling）」というルーター構成では、汎用トークンによる信号の希釈が起き、ルーティング確信度が99%に達しない（0.6〜0.7に留まる）という課題が判明しました。

これを解決するため、次回は**「どのようなルーター構造が、パラメータ数を極小に抑えつつ最も高い確信度（>0.99）を出せるか」**を検証するベンチマーク（Notebook 24）を作成します。

### 🎯 次回検証するルーターのバリエーション案
- **アプローチA (Last-Token Routing)**:
  全トークンの平均ではなく、自己回帰モデル特有の「文章全体の文脈を圧縮した最後のトークン」のベクトルのみをルーターへの入力とする手法。
- **アプローチB (MLP Router)**:
  単層のLinearを廃止し、表現力を高めるために非線形な隠れ層を持たせた小さな多層パーセプトロン（`Linear -> ReLU -> Linear` など）を採用する手法。
- **アプローチC (Attention-based Pooling Router)**:
  各トークンの重要度をAttention機構で重み付けし、「計算に関係しそうなトークン」にのみフォーカスしてルーティング確率を算出する手法。
